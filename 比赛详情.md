基于 OpenClaw 的本地生活「全天候私人管家」
赛题描述
任务背景：旨在利用当下最热门的开源智能体框架 OpenClaw，通过其 IM 原生交互和强大的 Skill 扩展机制，重塑本地生活服务（餐饮、出行、娱乐）体验的技术挑战赛命题。

任务目标：展示基于 OpenClaw 的自然语言对话如何改变传统的本地生活 App UI交互。

任务描述：
1、框架要求：必须基于 OpenClaw 开源框架进行开发，可选用任意版本，但需在文档中说明。
2、Skill 开发：开发的 Skill 需严格遵循 OpenClaw 的插件规范（包含SKILL.md 定义与实现代码），且至少实现 3 个与本地生活相关的核心技能。
3、数据安全：不得收集或存储任何真实用户个人信息，所有用户偏好必须通过模拟数据或用户显式输入定义。

复杂指令下的多轮对话评测系统
赛题描述
任务背景：在履约数字人外呼场景中，系统自动发起与用户的通话，对话模型需要根据预设的指令完成具体的任务，由于指令中包含复杂的任务流程和多种约束，人工评估成本高且难以量化。需要构建一套针对对话模型任务指令遵循效果的可解释可量化的自动评估能力。

任务描述：设计并实现针对已有任务指令的自动化指令遵循能力评估系统，需要保证评估过程的可解释性以及评估结果的可靠性。

交付目标：
1、构建用户模拟器，能够充分有效测试对话模型在特定任务指令下的效果。
2、自动产出评测报告，要求评测过程可解释结果可量化。（脱敏数据将在报名后一个工作日内发至队长邮箱）

03
美甲AI试戴与智能运营
赛题描述
任务背景：在美甲服务场景中，让用户"看得见效果"和帮助运营"抓住流行趋势"是两大核心痛点。用户在浏览平台数百款美甲图片时，无法想象实际上手效果，担心肤色不搭、手型不配，线下试戴又需要预约到店成本高，最终导致决策周期长、放弃率高。运营侧面对海量款式库存，人工统计数据费时费力且更新滞后，对热门款式的识别往往慢几天错失最佳运营窗口，推荐策略粗放导致转化效果不佳。当用户无法"所见即所得"验证效果，运营无法"实时感知"市场变化时，即使款式本身优质，也会因体验断层和信息滞后而流失用户和错失商机。

任务描述：赛题聚焦把"AI试戴"和"智能运营"做成可落地、可评测、可迭代的完整系统。

交付目标：参赛者需要让用户能够快速看到款式试戴效果做出决策，同时让借助openclaw等AI助手能够实时监控款式热度、分析流行趋势、自动生成运营策略，以提升用户转化率和运营效率。（脱敏数据将在报名后一个工作日内发至队长邮箱）

04
AutoSolver：让 AI Agent 自主求解配送分配问题
赛题描述
任务背景：在外卖配送系统中，高效地将配送任务分配给骑手对于运营成功至关重 要。本题描述了一个真实场景：配送平台需要将多个配送订单分配给可用骑手，同时需综合考虑配送时间、距离、骑手意愿，以及合单配送（将多个订单合并给同一骑手）的可能性。同时系统允许拒绝订单，并且对骑手接起对应订单的概率有预测；系统允许一个订单同时指派给多位骑手，最先接起订单的骑手获得订单。系统需要在多个目标之间取得平衡：最大化接单订单数量、最大化系统收益（以分数表示），并通过智能合单来优化资源利用率。

任务描述：给定一组配送订单（任务）和可用骑手，以及每个任务-骑手组合的预计算分数，求一个最优分配方案，使接单订单数量最大化，同时总分数最小化。

交付目标：选手需要构建一个 AutoSolver Agent 系统，该系统具备以下能力：
1、自主策略探索：Agent 能自主提出并尝试不同的求解策略（贪心、ILP、启发式、LLM 直接推理等），无需人工指定用哪种算法。
2、自动评估与筛选：每次尝试后，系统自动计算目标分数，判断是否优于当前最优解，决定保留或丢弃。
3、迭代改进循环：系统能根据历史实验结果，自主调整下一轮策略（例如：发现贪心在合单场景表现差，则自动转向其他方向）。
4、 输出最终解：在时间限制内（10 秒/测试用例），输出当前最优分配方案。

05
现在就出发：AI本地路线智能规划
赛题描述
任务背景：用户在出行或游玩时，往往需要将多个目的地串联成高效路线，同时面临"想吃好又不想排队、时间预算二选一"等多重决策负担。现有搜索和推荐体验需要用户反复筛选、自行组合，决策成本高。本课题探索如何利用大语言模型，结合点评 POI 数据、UGC 智慧和用户个性偏好，自动生成可直接执行的个性化路线方案。

任务描述：设计并实现一个本地智能路线规划系统，完成以下流程：用户输入游玩/出行目标，系统结合 POI 数据与服务、用户评价语料等，自动生成多维度最优路线方案，支持按时空、偏好等约束动态调整。

交付目标：
1、路线生成：能够根据用户意图自动串联多个 POI，生成完整路线安排。
2、多条件与个性化：满足差异化条件约束结合用户历史偏好生成差异化方案。

06
本地探索：周末闲时活动规划
赛题描述
任务背景：
周六上午9点，小明迎来了难得周末双休，他给美团发了一条消息："今天下午是空的，想和老婆孩子/朋友出去玩几个小时，别离家太远，帮我安排一下。"
-家庭场景：孩子5岁，老婆最近在减肥
-朋友场景：总共有4个人，2个男生2个女生
-然后他就把手机递给了老婆 / 发给了朋友，看看这个安排好不好。

注意：这不是搜索推，这是「帮你把事情做完」，美团应该能在接下来的几分钟里：
-规划出一条「下午4-6小时」的综合方案，例如：去哪玩 → 玩完去哪吃 → 有没有额外活动
-查到有没有适合群体不同要求的餐厅，以及餐厅有没有位置，是否需要提前排队
-吃饭前后的活动安排，例如亲子乐园、和朋友的展览、citywalk小吃街等等
-将上述吃喝玩的方案完成可执行可落地的动作，小明确认方案后就可以一键安排所有下单/预约/送到指定餐厅蛋糕鲜花等操作
-把计划发给小张："搞定了，下午 2 点出发，先去……"


任务描述：
构建一个本地场景短时活动规划与执行Agent，接受一句自然语言目标，输出可执行的完整方案并自动完成关键下单/预订动作。

交付目标：Demo（命令行/Web UI 均可）
1、完整的Tool实现代码（包含 Mock API 调用）。
2、设计文档（≤2 页，说明 Planning 策略、工具调用链路、异常处理机制）。
