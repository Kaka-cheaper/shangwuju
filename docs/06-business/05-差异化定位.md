# 差异化定位

> 一图回答：「为什么用户要用晌午局，而不是已有产品？」

---

## 一、四象限对比

```text
                  「主动决策」
                       ▲
                       │
      ChatGPT（通用）   │   晌午局（强约束 + 一键执行）
                       │
   被动     ────────────┼──────────── 主动
   信息                 │             下单
                       │
   小红书（看别人玩）    │   美团团购（已知商品下单）
   大众点评（找店）      │
                       │
                  「被动浏览」
```

晌午局占据 **「强约束 → 主动下单」象限**，是目前没有公司在做的位置。

---

## 二、与各家逐一对比

### vs 大众点评

```text
| 维度        | 大众点评                  | 晌午局                              |
|------------|--------------------------|------------------------------------|
| 入口        | 类目搜索 + 列表筛选        | 一句话（自然语言）                   |
| 决策机制    | 用户自己看评分 + 排队       | Agent 直接出方案 + 异常自动重规划    |
| 输出        | 一家店                    | 六段行程（出发 / 主活动 / 用餐 / 返回）|
| 多约束处理  | 多次筛选漏斗               | 一次性吸收所有约束（年龄 / 距离 / 减肥）|
| 转发        | 一张店铺图                 | 一段为亲友写好的文案                  |
```

**核心差异**：大众点评是「我已经知道想吃什么类型的店，找推荐」；晌午局是「我不知道怎么安排下午，给我一份可执行清单」。

### vs 美团团购

```text
| 维度        | 美团团购                  | 晌午局                              |
|------------|--------------------------|------------------------------------|
| 用户心智    | 我要买这个商品/团购券      | 我要安排一段时间                     |
| 商品颗粒    | 单店 / 单品                | 多店组合（活动 + 餐饮）               |
| 约束类型    | 价格 / 地点              | 价格 / 地点 / 时长 / 多人各自的偏好    |
| 路径长度    | 商品页 → 下单（短）        | 对话 → 出方案 → 确认 → 多店执行（长）  |
```

**核心差异**：美团团购解决「我要买这个」，晌午局解决「我要做这件事」。两者错位互补，**不抢同一笔交易**——用户在晌午局确认行程后跳转美团下单是天然衔接。

### vs 小红书

```text
| 维度        | 小红书                    | 晌午局                              |
|------------|--------------------------|------------------------------------|
| 内容形态    | UGC 笔记（看别人玩什么）   | AI 直接为我出方案                   |
| 决策成本    | 看 N 篇笔记自己拼凑        | 一次对话                            |
| 个性化      | 推荐流（你可能感兴趣）     | 强约束（5 岁孩 + 减肥老婆 + 距家 5km）|
| 时效性      | 笔记可能 2 年前的店         | 实时容量（β 阶段）                  |
| 转化路径    | 笔记 → 站外搜店 → 下单      | 对话 → 行程 → 一键执行                |
```

**核心差异**：小红书是「灵感库」，晌午局是「方案输出器」。用户从小红书获得的**灵感**最终需要一个像晌午局这样的工具落地成**可执行清单**。

### vs ChatGPT / 通义千问 / 文心一言

```text
| 维度          | 通用 LLM                | 晌午局                              |
|--------------|------------------------|------------------------------------|
| 知识时效      | 训练截止时间               | 实时数据（高德 + 大众点评 + 直签商家）|
| 本地化        | 通用全球                  | 本地生活强约束                       |
| 数据接入      | 不接 / 浅接                | 深度接入 8 个领域 Tool               |
| 行动闭环      | 给建议（你自己去预约）       | 一键预约 + 购票 + 转发                |
| 异常处理      | 不知道餐厅满 / 票售罄       | 显式触发 + 自动重规划                 |
| 个性化        | 不记忆 / 仅 chat 内         | persona + memory 累积学习             |
```

**核心差异**：通用 LLM 给「建议」，晌午局给「方案 + 执行」。前者解决「想知道」，后者解决「想做到」。

## 三、唯一差异化标签

> **晌午局 = 唯一一个「对话即下单」的本地生活 AI Agent。**

「对话即下单」拆解：

```text
对话    自然语言、强约束、跨多领域（吃 / 玩 / 行）
即      跨 turn 反馈重规划、个性化记忆、异常自愈
下单    可执行（reserve / buy_ticket）+ 可转发（一段文案带亲友）
```

这不是单一功能，而是**入口形态的改变**：用户从「列表 + 筛选」习惯，迁移到「描述需求 + 等结果」习惯。

## 四、为什么我们能做到 / 别家做不到

```text
| 障碍                | 现有玩家为什么过不去                    | 晌午局为什么能            |
|--------------------|--------------------------------------|--------------------------|
| LLM 成本控制        | 通用 LLM 调用过多                      | 8 Tool + 规则编排，调用确定 |
| 多约束推理         | 通用 LLM 在多约束下漂移                | 意图解析 + 规则强约束          |
| 本地数据接入        | 平台之间互锁数据                       | 第三方角色 + 多源混合          |
| 异常处理机制        | 列表筛选模型不需要                     | Agent 范式天然能力             |
| 个性化              | 列表式产品记不住偏好                   | persona + memory 双驱动        |
| 执行闭环            | 通用 LLM 不接生活服务                   | 直签商家 + 大众点评 / 美团 API |
```

每一项壁垒不是单点的，是「LLM + 数据 + 商业模式 + Agent 工程」的复合能力。

## 六、风险：差异化会不会被快速复制？

```text
风险点                          应对策略
LLM 调用本身不是壁垒              壁垒是「Agent + 数据 + 商业」的组合
通用 LLM 厂商可能下场              我们的优势是本地生活领域 know-how + 直签商家
美团自己做                       重叠但错位（团购侧难做对话入口）
开源方案出现                     我们靠数据 + 商家关系建立护城河
```

**核心：差异化不是技术，是「场景 + 数据 + 商业」三角的复合**。
