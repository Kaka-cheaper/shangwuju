# 商业模式

> 三个候选模式 + 一条推荐路径 + 单位经济假设。

---

## 一、商业模式候选

### 候选 1：流量分发（餐厅广告位）

类比高德地图、美团点评。商家付费买「Agent 优先推荐」位次。

```text
| 项                  | 说明                                              |
|--------------------|--------------------------------------------------|
| 收费形式            | CPC（按点击）/ CPM（按曝光）/ 包月席位            |
| 主要客户           | 中型连锁餐饮、本地特色门店                        |
| 利润空间            | 行业基准 30-50%（与流量平台同档）                  |
| 收入弹性            | 跟着 DAU 线性放大                                 |
| 风险                | 推荐质量稀释 → 用户流失（必须设广告位上限）         |
```

**实施红线**：广告位 ≤ 推荐结果的 20%，且必须显式标注「赞助」。Agent 在意图无法被广告候选满足时仍走自然推荐。

### 候选 2：订阅制

```text
| 项                  | 说明                                                |
|--------------------|----------------------------------------------------|
| 定价                | ¥30 / 月（参考国内 SaaS 个人订阅基线）               |
| 解锁权益            | 无限次规划 / 实时家庭日历同步 / 多人并行规划 / 历史方案搜索 |
| 主要客户           | 高净值家庭（亲子 / 老人陪伴 / 商务接待）              |
| 利润空间            | 净订阅收入扣除 LLM 成本（~ ¥3 / 月 / 用户）           |
| 收入弹性            | 跟着付费用户数 × ARPU 增长                          |
| 风险                | 需要长期可感知价值，不是一次性体验                   |
```

> 定价点位为 2026-05 推算的产品规划假设，不是市场实测数据，正式上线前以用户访谈为准。

### 候选 3：抽佣

```text
| 项                  | 说明                                                |
|--------------------|----------------------------------------------------|
| 抽佣比例            | 5%（商户预约）/ 10%（活动门票，参考行业基线）         |
| 主要客户           | 直签商家、本地活动主办方                             |
| 利润空间            | 与餐厅 / 票务平台错位（我们做规划入口，不是下单链路）|
| 收入弹性            | 跟着 GMV 放大                                       |
| 风险                | 早期 GMV 体量小、与美团重叠（需要错位）               |
```

**与美团错位的关键**：美团是「我已知商品下单」，晌午局是「我未知场景规划」。同一家餐厅在美团是「单店搜索 + 团购券」，在晌午局是「老婆想吃健康简餐 + 距家 5km 内」自动推荐。**入口角色不同，不抢同一笔交易**。

## 二、推荐路径

```text
0 ~ 3 月  完全免费验证 PMF
          目标：100 + DAU、30%+ confirm 率、≥2 个商家直签
          关键：用户主动分享行程文案 → 自然增长

3 ~ 6 月  引入候选 1（流量分发）
          目标：MAU 1k、ARPU 0.5 元（流量分发收入 / MAU）
          关键：推荐质量监控不下降 + 广告位标注合规

6 ~ 12 月  叠加候选 3（抽佣）+ 候选 2（订阅）小流量验证
           目标：MAU 10k、ARPU 2-3 元（流量 + 抽佣综合）
           关键：直签商家覆盖头部 5 商圈 + 高净值用户订阅渗透 5%
```

为什么先免费 → 流量 → 抽佣 + 订阅：

```text
1. 免费阶段把 PMF 跑出来，避免「过早商业化伤体验」
2. 流量分发是「弱侵入」收入，不破坏 Agent 决策中立性
3. 抽佣 + 订阅是高 ARPU 但难起步，等用户基数到位再上
```

## 三、单位经济（Unit Economics）

> ⚠️ 以下数字均为**规划假设**，非实测数据。正式上线前以小流量灰度数据为准。

### 假设输入

```text
| 参数                       | 假设值          | 依据                          |
|---------------------------|----------------|-----------------------------|
| 单 session LLM 成本         | ¥0.20          | DeepSeek 单价 + 平均 token 量 |
| 单 session 高德 API 成本   | ¥0.05          | 5 次查询 × 单价               |
| 单 session 总成本（仅 AI）  | ¥0.25          |                              |
| 月人均 session 数（活跃用户）| 8 次           | 假设每周 2 次（周末 + 节假日） |
| 月活留存                   | 30%            | 同类产品基线                  |
| CAC（用户获取成本）         | ¥10（异业合作）| 详见 06-增长路径.md             |
```

### 候选 2 订阅制单位经济

```text
ARPU（月）        = ¥30 × 5% 渗透率（高净值）= ¥1.5
变动成本（月）    = ¥0.25 × 8 = ¥2.0
注意               单纯订阅 5% 渗透不够覆盖成本，必须叠加流量分发
```

### 候选 1 流量分发单位经济

```text
ARPU（月）        = ¥1.0（行业基线）
变动成本（月）    = ¥2.0
GP（月人均毛利）  = -¥1.0
```

### 综合（流量 + 抽佣 + 订阅 5% 渗透）

```text
ARPU（月）        = 1.0（流量）+ 0.4（抽佣）+ 1.5（订阅）= ¥2.9
变动成本（月）    = ¥2.0
GP（月人均毛利）  = ¥0.9
LTV（24 月留存）  = ¥0.9 × 留存衰减积分 ≈ ¥10
LTV / CAC         = 10 / 10 = 1.0  ⚠️ 临界
```

### 关键改善路径

```text
1. LLM 成本 ¥0.25 → ¥0.10：缓存 + Tool 选择确定性 + 自研轻量模型
2. ARPU ¥2.9 → ¥5：直签商家拿到优惠位，提升抽佣 GMV
3. CAC ¥10 → ¥5：用户分享文案带来自然流量（一句话方案的传播性优于商品列表）
```

改善后 LTV / CAC ≈ 4-6，符合可持续增长门槛。

## 四、与美团生态的衔接

### 4.1 业务侧应用（被收购 / 战略合作可能性）

晌午局做的是「**对话即下单**」入口，与美团既有产品形成互补：

```text
| 美团既有                     | 我们的角色                     |
|----------------------------|------------------------------|
| 美团团购（已知商品下单）     | 规划入口（未知场景 → 商品）   |
| 大众点评（看评价）           | 决策辅助（评分 + 实时容量混合）|
| 美团 AI 助手（搜索向）        | 主动 Agent（场景 + 约束驱动）  |
```

衔接点：

```text
- 数据：晌午局 Agent 调大众点评 / 美团 POI API（合作或开放）
- 流量：用户在晌午局确认方案 → 跳转美团下单（CPC / 抽佣）
- 用户：晌午局做「家庭场景」深度，美团做「日常下单」广度
- 模型：晌午局沉淀的「场景 → 商品」映射数据反哺美团 AI 推荐
```

### 4.2 战略价值

```text
对美团：
  + 抢到「场景化 AI 入口」时间窗口（vs 字节豆包 / 百度文心生态）
  + 沉淀本地生活场景化数据（团购侧拿不到）
  + 验证 Agent 商业化路径（小流量验证 + 大流量复制）

对晌午局：
  + 借美团数据合规与商家关系起步
  + 用户量级跨越式增长
```

## 五、可解释的财务模型示例

```text
假设（保守）：
  6 个月 MAU = 10k
  ARPU（综合）= ¥2.9 / 月
  LLM + API 成本 = ¥2.0 / 月
  人力 + 服务器 = ¥80k / 月

月收入 = 10k × ¥2.9 = ¥29k
月毛利 = 10k × (2.9 - 2.0) = ¥9k
月净亏 = ¥9k - ¥80k = -¥71k

→ 6 个月需 ~¥430k 现金 buffer
→ 12 个月跑到 MAU 50k 时月毛利 ¥45k，已能覆盖大半人力
→ 24 个月 MAU 200k 时已跨越亏损线
```

> 上述数字是规划假设，不构成投资建议。Hackathon Demo 阶段无成本压力，本节仅证明「商业模型可推演 + 路径合理」。
