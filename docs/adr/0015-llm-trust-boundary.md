# ADR-0015: LLM 信任边界——自由域、确定域与词表缓存域

- **日期**: 2026-07-04
- **状态**: Accepted
- **范围**: 全后端决策点归属(路由/意图/工具/规划/求解/换菜/叙事/确认/房间/会话基础设施)——不新增功能,把散落在 ADR-0010/0011/0014 与四份专项调研里的分界决策收束成一条可执行原则 + 不变式清单 + 演进门控。
- **承接**: [ADR-0011](0011-llm-first-routing-obligations.md)(一脑三壳)、[ADR-0010](0010-multi-activity-toptw-planning.md)(TOPTW 求解)、[ADR-0014](0014-requirement-analysis-provenance-and-hard-constraints.md)(出处与硬软约束)、[L0 响应义务契约](../L0-响应义务契约.md)。

## 背景

这是当下 agent 架构的头号路线之争:**自主工具代理**(一个 LLM 循环自己决定查什么调什么)vs **显式工作流**(路径固定,LLM 只在语义决策点上岗)。Anthropic 的工程指南(Building Effective Agents, 2024-12)站位明确:任务结构已知用工作流,能用工作流就不用代理——自由度的价格是延迟方差、循环失控、注入面扩大、决策过程不可审计。

本仓的实证把两个方向的失败模式都钉死了:

- **全靠规则会错**:确认词表单字「行」撞上「银行卡里没钱了」「这个行程安排的,还行吧」判成确认;强反馈词「太久」把「太久没回复我了,人还在吗」送去重排;画像线索「还记得我」抢答「你还记得我们之前聊了什么」(答非所问)——语义是开放集,词表永远追不完(勘察/点火/I 类探针程序化实锤,修于 9eecef0)。
- **全靠 LLM 也会错**:TravelPlanner(ICML'24)实测 GPT-4 级模型当自由代理做带约束行程规划,最终通过率 **~0.6%**——组合优化不是语言模型的活。本仓 stub 实验另证:类型污染(dict 代替活对象)能连兜底规划器一起击穿(spike_room_resume.py 实录)。

2026-07-04 的全后端普查(约 120 个决策点逐一核归属,file:line 台账见普查报告)确认:边界真实存在且执行良好——九成以上归属正确,错位 11 处全部有修法。本 ADR 把这条边界成文。

## 决策

### 1. 分界原则(一句话判据)

**无标准答案的语义判断归 LLM;有标准答案的计算、校验、查表归确定性代码;两域之间用类型化契约(schema)交接。**

- 归 LLM 的:路由义务分类、意图抽取、反馈合并、蓝图选点(哪里/做什么/多久)、权重品味、叙事措辞、隐晦软约束、弃答文案。
- 归代码的:时间游标与通勤估算、critic 全部判据(数量见 `agent.planning.critic.validate.REGISTRY`,现场计数、不硬编)、ILS 目标函数与调度、硬软放宽序、预算比较、桌型分档、台账 merge、版本志、出处 diff。
- 契约:IntentExtraction/Blueprint/RouterDecision 等 Pydantic schema,词典 Literal 镜像数据目录可查标签集——**schema 即解耦接口**,不是耦合。

### 2. 不变式:四条永不交给 LLM

| # | 域 | 理由 | 证据锚 |
|---|---|---|---|
| 1 | **行程求解核心**(ILS/调度/修复闭环) | 组合优化 LLM 通过率 ~0.6%(TravelPlanner);LLM-Modulo 范式=LLM 提案+求解器保可行 | `ils_planner.py` 全文,ADR-0009/0010 |
| 2 | **出口审计**(exit_audit) | 审计者不能是被审计者;逐字段满足度比对有标准答案 | `critic/exit_audit.py`,挂点 `finalize_plan.py` |
| 3 | **壳1 注入检测** | 「是不是攻击」不能交给可能正被这次攻击劫持的模型 | `injection_detector.py`(fail-open+L2/L3 纵深) |
| 4 | **确认执行**(pending_actions 忠实回放) | 执行期零决策是全链路防编造的收口——确认时不读 intent、不重新决策,只回放规划期算好的清单 | `execute_finalize.py:116-233` |

任何未来收编/工具化不得触碰这四条。

### 3. 第三类活:词表 = 物化(缓存)的语义判断

普查发现分界不是二分:canonical 等价表、嗅探规则表、QA 线索表这类**语义判断因延迟/稳定性被物化成确定性词表**的存在物,既不"拿得准"也没交给 LLM。承认这一类,并立三条纪律:

1. **单一真相源**:词表是缓存的语义判断,必须有 SoT 声明与来源纪律(范本:`category_vocab` 等价表;反例:parser 的 list-tag 字面校正未复用嗅探词表,已记档);
2. **漏配失败方向预先设计**:词表没盖住时往哪边倒必须显式选择——宁少断言、落 LLM、或落保守地板,绝不默默做假断言(范本:未满足检测 fail-safe「宁缺毋误报」);
3. **多处消费共享实现**:同一词法判断被重排与告知两处消费,必须同一把尺子(范本:`poi_desire_match`)。

**精度契约**(9eecef0 成文):规则**拍板**层(命中即路由、无兜底)每条词目必须单独接近百分百精度——判据是"能想象一个日常语境里它出现但用户不是这个意思,就剪";召回损失归脑子(慢 2-4 秒但带上下文)。高召回**粗筛**层(不拍板,仅辅助)不受此约束。

### 4. 接缝模式:生成归 LLM,验收归代码

类型化契约拦得住**结构漂移**(字段缺失/类型错/枚举外值),拦不住**结构合法但事实错误**的自由文本(note 里的时刻、叙事里的数字、摘要里的隐私)。凡 LLM 自由文本进入确定性消费或直接展示,必须有验收步:

- 数值类:代码预格式化喂给 LLM 照抄,或产出后 clip/强制覆盖(范本:refiner 时长一致性强制、偏好分 clip[0,1]);
- 引用类:白名单校验,失败整体回落(范本:cta_chips send 白名单、node_chips 全量校验);
- 事实类:与真相源交叉核验(修缮批:note 时刻 ∈ 真实可订槽);
- 合规类:代码后验(记档:记忆摘要脱敏 regex 验收);
- 存活类(2026-07-11 四条不变式批新范本):**缺键继承守卫**——refiner 对 intent 是整体替换,不在 LLM 输出里的字段被 Pydantic 默认值静默顶上;区分「键不存在=LLM 忘写→从 original 继承」与「显式输出反值=用户改口→放行」,few-shots 全量带字段是第一道防线,代码守卫是第二道(范本:`refine_intent` 的 `explicit_dining_requested` 守卫,C5a);
- 必说类(同批新范本):**代码后置追加**——「必须说且数字必须对」的句子不进 LLM 的活,narration 定稿后由代码构造追加,构造即正确、零验收机器,LLM/模板两路径共用一个追加点(范本:出发时刻分歧强制披露句,C3;prompt 织入+substring 验收是升级路径,归 4 拍重写批评估)。

### 4b. 显式保形原则(2026-07-11 四条不变式批升格成文)

**归一化不得吞显式**:抽取/合并层把用户明说的信息归一成系统词汇时,显式信息的「形」不能丢——丢形=下游永远无法区分「用户说的」与「系统猜的」,I3(显式诉求零丢失)在入口就被击穿。这不是新发明,是把既有零星实践(`_detect_out_of_vocab_social` 安全网、G-1 距离披露、G-3 定性预算不编数)升格为显式原则。**五次实例**(同一坑五次撞见,含一次撞见在新设计自身):

1. **吃饭泛词**:「找个地方吃饭」无品类可归一 → 曾被静默丢弃(preferred_poi_types 只承接品类词)→ tristate 字段承接(C4/C5a);
2. **refiner 缺键**:反馈轮整体替换 + few-shot 不含新字段 → 显式诉求被默认值静默翻转 → 缺键继承守卫(C5a);
3. **钟点 token**:「晚上七点」被归一成 today_evening → 声明钟点丢形,披露句永不触发 → parser 保形规则(C8/T2);
4. **终点表达**:「十点前得回家」被折算成 duration 区间 → 显式门禁无处安放,且与 fold 相互作用放大(折叠保时长后移终点=打破门禁)→ ambiguous_fields 自报+到家时刻披露(C8/T2),return_by 一等字段挂路演后;
5. **tristate 默认值**:bool=False 把「没提」和「明确不要」折叠成同一个值——归一化吞显式发生在设计自身 → Optional[bool]=None 三态(C4)。

新增 intent 字段的配套纪律见 `schemas/intent.py` IntentExtraction 类 docstring(必同步 refiner few-shots + 评估缺键守卫)。

### 5. 置信度纪律

LLM 自报置信度不可校准(选择性分类文献共识)。**只允许向保守方向消费**(<0.6 降澄清),永不作为向激进方向升级的依据(如"高置信就跳过校验")。当前实现符合,此条防未来破坏。

### 6. 演进阶梯(每级有门,不齐步走)

| 级 | 内容 | 开闸条件 |
|---|---|---|
| 1 | 打包器画像并轨(persona 标签+累积偏好+近期行程进路由脑子上下文) | I 类元对话探针真实摸底显示"数据缺失答不出" |
| 2 | E-2-d 收编四步(确认/预约词表 → 画像问答 → 方案问答检测层 → 强反馈) | E-4 语料统计出该规则真实命中率与延迟代价 |
| 3 | 只读工具化(查画像/查历史/查方案成为脑子可调工具,QA 规则层退役) | 长尾问答类型持续膨胀,且并轨后仍答不好 |

**写操作(下单/改方案)永不进工具循环**——与决策 2 不变式 4 同源。

### 7. 身份边界(2026-07-05 增补,用户拍板)

demo 无账号体系,**会话即身份**:预置画像(persona 标签/预算先验)是**共享只读模板**,按画像 id 键控;累积记忆(行程档案+偏好累积两轨)是**会话私有**,按 session_id 键控(房间=collab_{room_id}),进程内存储。多访客并发串味由此根治;生产迁移=键从会话 ID 换账号 ID,机制不动。证据锚:6092223(读写分离批,消费方全量勘察含任务书外 5 处;探针 66→67,I4/I5 语义翻转"跨会话诚实不认识"+I6 同会话记忆延续钉死)。规划层的身份数据入口唯一(意图抽取,物化为带出处意图字段);路由脑子的身份数据补齐(读侧并轨)维持阶梯级 1 门控。

## 备选已弃(Alternatives considered)

- **全自主 agent**:TravelPlanner 0.6%;决策链变聊天记录,出处/台账/审计不可能;工具结果回灌是间接注入重灾区;演示延迟方差不可预算。
- **全规则**:四例实锤 FP;开放语义追不完;ADR-0011 已判此路线死刑。
- **LLM 自由输出字段**(开放 schema):字段无人消费=假接受(ADR-0014 全篇在消灭的东西);出处四值/出口审计都建立在字段集已知上;查询侧词典镜像数据目录,发明的字段无物可查。

## 不做清单
