# ADR-0017: 首段等待折叠与出发时刻语义

- 状态：Accepted（2026-07-11，四条时间线不变式批 C2/C3 落地）
- 范围：拼装层（assemble）+ 口播（narrate）+ 意图卡语义
- 上游：《时间线可信四条不变式综合实现方案》（十轮迭代终稿）；I1 权威原文——
  「出发时刻回答『我几点出门』；自由休息只许出现在中段、且讲得出因由；
  口播报的时长必须是体验时长」

## 背景（罚站 bug，用户亲手截图的台面问题）

正向游标 + 餐厅槽吸附的组合把等待放在**到达之后**：19:00 出门 + 3 分钟通勤
→ 19:03 到店，19:00 槽已过、19:30 满座 → 吸附到 20:00 落座 → 57 分钟「自由
休息」排在店门口。机制正确地做了错误的事——assemble 的不变量（`to_start >=
hop_end + buffer`）从未被违反，等待只是被放错了位置；`ItineraryCard` 如实
渲染成 gap 行，直接暴露在演示时间轴上。

这在排程理论里是 **backward scheduling from anchor**（frePPLe 式：先按到达/
预约锚点倒推出发时刻，倒推不可行再退化为正推）的缺位。本 ADR 记录路演前落地的
两个切片及其带来的 **「出发时刻」语义变更**：
- **1.3-B 首段折叠**（`_fold_leading_wait`）：只吸收家→首站的等待，先行落地；
- **1.3-A 完整倒推**（`_find_backschedule_anchor` / `_backschedule_departure`，
  2026-07-11 追补落地，见文末更新）：对任意位置的餐厅硬槽位锚点回推出发时刻，
  消灭**中段**死等（情侣看展 15:45→17:00 的 75min 空洞是驱动此追补的真机病灶）。
  实现为"预演一次正向 assemble → 倒推调整 preferred_start_time → 重跑一次"的
  外层循环，cursor 单调内核零改动（方案 1.11-c）；首段折叠成为"锚点=首站"的
  退化子集，两者共存不冲突。

## 决策

### 1. 首段等待折叠进出发时刻（C2）

`assemble_from_blueprint` 尾部后处理 `_fold_leading_wait`：首 hop 结束
（+buffer=0）与首个活动节点 start 之间的正向差额 g，整体后移 `nodes[0]`/
`hops[0]` 的 start_time 吸收；`total_minutes` 从折叠后出发时刻起算（字段语义
不变，虚胖如实缩水）。**出发时刻从此是「系统计算值」而非「蓝图声明值」**——
这是本 ADR 存在的理由（语义变更必须留痕）。

- **一处覆盖三路径**：LLM / ILS / rule 全部经 `assemble_from_blueprint` 拼装
  （ILS 见 `ils_planner` 的 `route_to_blueprint → assemble` 调用链），无需
  各路径独立改动。
- **只折首段**：首段差额的唯一物理语义是「出门太早」（首跳 buffer=0，差额无
  其它成因），后移出发无损；中段 gap 可能有正当因由（等座/消食），归完整
  倒推批。
- **单轮收敛 + 幂等（实证钉死）**：吸附规则「不早于自然到达的最早可用槽」
  以 `>=` 判定——折叠量 g 恰使新自然到达 == 原选中槽，等号复选同槽，一轮
  收敛不振荡；再折恒 0（`test_assemble_fold.py` F2/F3）。
- **不制造新违规**：只动 n0/h0 两个时刻且保持精确对齐，活动节点/槽位/窗全部
  不动 → critic 各 check 输入不变或更优；critic 修复闭环的收敛性与折叠前
  同构。**止振结构论证**（方案 1.24）：「折叠缩短 → 时长不足 → 修复加节点」
  回路第一跳不存在——时长不足是 SOFT（ADR-0010 决策 10「短而好不打回」），
  修复闸门 `route_after_critic` 只认 HARD；折叠对唯一 HARD 方向（超长）是
  单调改善。
- **释义（release time）**：口语的「7 点出门」默认是**最早可出发时刻**，不是
  「必须 7 点整迈出门」的承诺——折到 19:55 仍满足「≥19:00」；I1 权威原文的
  例子（19:55 出门 20:00 落座）本身就是折叠。
- **折叠量 ≥90 分钟**（拍板项 P1）写 `quality_issues`：巨额差额说明规划没填
  满时段，问题在蓝图/涌现层——折叠不设硬上限（诚实优先）但异常不静默。
- **可观测性**（方案 1.29，≤10 行）：assemble 内 `logger.info` 一行 +
  finalize_plan 现算折叠量追加进 `decision_trace.blueprint_rationale`
  （state 同时有 blueprint 与 itinerary，差值现场可算，零 schema 零管道）。

### 2. 出发时刻分歧强制披露（C3）

意图卡渲染 `intent.start_time` 声明值（「你说的」），时间轴渲染折叠后
`nodes[0].start_time`（「计划的」）——声明含明确钟点且折叠值更晚时，**必须
披露，不分大小**（原 ≥15min 阈值作废，方案 1.16 升格）。实现走「代码后置
追加」（拍板项 P9，ADR-0015 决策 4 新范本）：narration 定稿后由代码构造追加
「你说 19:00 出门——其实 19:25 出发正好赶上，不用干等；想按原时间出门先在
附近转转也行，跟我说」，LLM/模板两路径共用 narrate_node 一个追加点，构造即
正确。**措辞含出路**（拍板项 P2）：把改写呈现为服务，给用户顶回的通道。

### 3. 意图卡保留声明值（九视图取值审计，方案 1.23）

九个视图逐一查实：唯一显示声明值的是意图卡（语义即「你说的」，与披露句配对
成立）；时间轴首行/口播/refiner 反馈轮摘要/房间快照显示折叠值；地图/决策
trace/确认执行流/版本志没有出发时刻位——无第三个漏改视图。SSE 无闪变：
`emit_assemble` 思考句不带钟点（docstring 红线已钉），首个携带行程 HH:MM 的
事件是 finalize_plan 之后的 ITINERARY_READY，拓扑上必然晚于折叠；blueprint
rationale 残洞用 prompt 纪律封（「rationale 不写具体钟点」，方案 1.34-W3）。

## 备选已弃（Alternatives considered）

- **前端容忍时刻更新**（渲染层吸收闪变）：无闪变已由事件序结构保证，前端
  改造是为不存在的问题付费；且把「等待放错位置」留在数据层，与地图/卡片
  互证时仍穿帮。
- **user_stated 出发时刻豁免折叠**（首段插「有因由的自由活动」行）：I1 权威
  原文明令「自由休息只许出现在中段」——豁免方案恰恰违反字面；且「就是想
  19 点出门晃悠」与「随口说了个整点」超出可靠解析能力，豁免会把大量随口
  整点误判成故意松弛，把罚站还给多数人。采用「首轮照折 + 强制披露带出路；
  用户顶回时豁免」的三段式（方案 1.19），豁免所需的 `departure_time_firm`
  状态位挂路演后（顶回是低频路径；已知限制：连续坚持会再次被折叠）。
- **完整倒推（1.3-A）已于 2026-07-11 追补落地**（原判"推迟路演后"被真机病灶
  推翻——情侣看展是主演示局，75min 中段死等违背 I1「本地人 3 秒常识审视」，
  不可带上路演）。落地形态即前述外层循环：预演→倒推锚点→重跑；首段折叠
  （1.3-B）如预期成为「锚点=首节点」的退化子集，无需回滚。**边界处理**：
  倒推出的出发时刻若早于原始 preferred_start_time 则维持原值、差额留作合法
  中段等待（`_backschedule_departure` 返回 `None`）——单锚点同一预演链路下
  此边界数学不可达，但保留契约并以纯函数测试钉住判断逻辑（未来若改用外部
  声明到达截止时刻做锚则变可达）。**范围限定**：只倒推时间序上第一个硬槽位
  锚点，多餐厅链路（晚饭+夜宵）下游锚点的 gap 保留——完整多阶段 VRPTW
  backward slack 传播挂路演后（演示场景无双餐厅局）。

## 边界（挂账，不在本 ADR 兑现）

- **终点锚盲区**（方案 1.25 重雷）：「十点前得回家」被折算成 duration 区间，
  fold 保持时长后移出发 → 到家时刻同步后移，可能打破用户显式门禁。8 个
  canonical 场景原文无终点表达（台面零暴露）；路演前最低限=parser 把终点
  表达自报进 `ambiguous_fields` + 到家时刻披露（C8/T2）；`return_by` 一等
  字段 + fold 终点护栏挂路演后。
- **`departure_time_firm` 豁免位**：挂路演后（见备选第二条）。
- **narrator 体验时长口径 / 中段 gap 因由标注**：与 1.3-A 强联动，同批挂账。

## 验收

- `test_assemble_fold.py`：折叠量精确 / 单轮收敛 / 幂等 / total_minutes 代数
  一致（critic 不覆盖的口播数字链专防）/ POI 首站零扰动 / 中段不折 / trace
  追加 / 质量信号 / 防御边界（11 例）。
- `test_departure_disclosure.py`：双值断言 / 无钟点负例 / 无分歧负例 / ISO
  提取 / narrate 集成正反例 / 防重复（8 例）。
- 元验收：时间线上每一个数字经得起本地人 3 秒常识审视——19:00 出门门口罚站
  57 分钟不再出现在任何视图。
