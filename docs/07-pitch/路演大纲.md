```text
小明的下午
─────────────
14:00  老婆说「下午带娃出去玩」
14:01  打开大众点评 → 翻 30 家店 → 不知选哪家
14:15  打开小红书 → 看 5 篇笔记 → 还是不会选
```

**痛点拆解**：

```text

|------------|--------------------------------------|
| 列表筛选     | 选项太多，决策瘫痪                     |
| 多人偏好     | 5 岁孩 + 减肥老婆 + 距家 5km 同时满足 |
| 异常情况     | 餐厅满 / 票售罄要重头再来              |

| 转发协调     | 还要给老婆描述清楚要去哪                |
```

**口播（45s）**：

> 半小时看 30 家店，老婆要减肥、孩子要玩、距家不能远，餐厅订位发现 17:00 还是满了。这不是个别情况，这是每个家庭周末的标准动作。

## 页 5：技术架构

```text
┌─────────────────────────────────────────────────┐
│  Web UI 层                                       │
│  Next.js 14 · SSE 流式 · 状态可视                  │
├─────────────────────────────────────────────────┤
│  Agent 编排层                                    │
│  ReAct 单一 Agent · 双 mode（rule + llm）          │
│  router → intent → planner → refiner → executor │
├─────────────────────────────────────────────────┤
│  Tool 层（8 个，OpenAI Function Calling 兼容）    │
│  search_pois / search_restaurants /             │
│  check_availability / route /                   │
│  reserve / buy_ticket / share_message /         │
│  user_profile                                   │
├─────────────────────────────────────────────────┤
│  Mock 数据层（Demo）                              │
│  39 POI · 45 餐厅 · 56 路线 · 5 persona            │
└─────────────────────────────────────────────────┘
```

**关键设计取舍**：

```text
1. ReAct 单一 Agent vs 多 Agent
   → 选单 Agent：Demo 可控、状态收敛于 messages
2. 规则 Planner + LLM 自由 Planner 双 mode
   → rule 是 Demo 安全网，llm 是评分加分
3. LLM 解耦（OpenAI 兼容 base_url）
   → DeepSeek / 通义 / OpenAI / 智谱 / Ollama 三件套切换
4. Mock 数据 vs 真 API
   → 题目硬性要求 + 抽象层就绪，MVP 切高德
```

**口播（45s）**：

> 我们用 ReAct 单一 Agent + 8 个 Tool 的范式，加一道双 mode 切换。规则模式是 Demo 安全网，LLM 模式跑出 Agent 自主决策的加分项。LLM 解耦——任意 OpenAI 兼容服务三件套切换。

## 页 7：上下文持久化 demo（评分项 1 + 5 加分）

> 评委关心：「Agent 不止是单 turn，跨 turn 才叫 Agent。」

```text
turn 1                     turn 2                    turn 3
──────────────────         ──────────────────        ──────────────────
用户：「带娃出去玩」          用户：「太远了希望      用户：[确认]
                                  3 公里以内」
   ↓                              ↓                        ↓
意图解析（注入 persona）   refiner 合并约束          executor 下单
distance_max_km = 5km     5km → 3km                  + memory 累积
                          复用主路径重规划            accepted_tags + 1
   ↓                              ↓                        ↓
8 Tool 编排                新候选只剩 1 家             下次同样输入
出方案 1                   出方案 2                   distance 默认值变 3km
```

**关键能力**（D9 边界严格遵守）：

```text
- persona prior 注入：5 个 mock user 的「我是谁」
- memory 累积学习：confirm 加分 / refine 减分（× 1.5 强惩罚）
- 反馈重规划：用户拒绝方案 → refiner 合并 → 复用主路径
- 输入域路由：「我累死了」/「你是谁」/「1+1=?」都能优雅响应
```

**口播（45s）**：

> 这页是我们的「哇时刻」——同一句话，不同 user 出完全不同方案；用户拒绝方案 + 自然语言反馈，Agent 自动合并约束重规划。这是聊天机器人做不到的闭环。
