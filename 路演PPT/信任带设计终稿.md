# 信任带设计终稿（AI 思考流）

> 定位：把原来散在左栏的三个技术面板（ToolTracePanel「Agent 思考链路」/ ThoughtPanel「Agent 在想什么」/ DecisionTraceCard「决策链路」）合成**一条**，改成"评委看着 AI 一路想下来"的第一人称思考流（DeepSeek 推理腔）。核心卖点：让评委*感受到*"它真听懂了人话、真在解题、还会自己揪错改掉"。
> 状态：设计定稿，待两处你终审（④⑤ 分种落词、header 标签）后落地。不动代码。

---

## 一、叙事逻辑（评委视角的三幕弧）

评委盯着看的 10-20 秒里，要读到一个故事：**听懂 → 干活 → 自省自愈 → 定稿**。高潮是"自省自愈"，它是和"套壳 ChatGPT"的分水岭。

关键原则（我这次想透的增量）：
1. **高潮必须两拍**——先亮问题（④），再亮修正（⑤）。只显示"已修正"评委无感；先"超时了"再"压一压"，评委才**看见它揪自己的错**。
2. **信息流要剪辑**——后端事件（15 条思考 + 3 次返工 + critic + fallback）不能直灌。选（15 条思考压成 3 拍）、并（3 次同样返工合成"发现→压→换引擎"的递进）、限（总量 ~7 拍）。
3. **忠实不编**（承接"叙事不编因"纪律）——每拍的具体词从**真实事件数据**生成，①③ 是真 LLM 输出，④⑤ 由真实违规类型驱动，不硬编一个没发生的故事。

## 二、七拍映射全表（事件 → 拍 → 来源 → 落词）

整条带统一**第一人称思考腔**（"让我…""用户…我理解成…"）。

| 拍 | 触发事件 | 类别 | 落词 |
|---|---|---|---|
| ① 理解 | `intent_parsed`（新 `understanding` 字段） | **LLM 现生成** | 例：*用户想周五晚和室友唱K，还说别太贵——我理解成想热闹、但钱别花太多*（风格红线见 §四） |
| ② 检索 | `tool_call`（search_pois/restaurants） | **固定句** | **让我先查询附近的店铺和时间** |
| ③ 规划 | 蓝图生成（新 `plan_reason` 短字段） | **LLM 现生成** | 例：*用户同行年轻人多，所以先用 KTV 带动气氛*（因人而异，LLM 每次真给；风格红线见 §四） |
| ④ 发现问题 | `critic_violations`（按违规码） | **分种模板** | 见 §三（超时那种＝**方案超出时间限制**） |
| ⑤ 修正 | `critic_fix_attempt` / backprompt | **分种模板** | 见 §三（超时那种＝**让我压缩一下时间**） |
| ⑥ 换引擎 | `plan_fallback`（→ ILS） | **固定句** | **还是不行，换成算法引擎** |
| ⑦ 定稿 | `done` / `itinerary_ready` | **固定句** | **规划成功** |

## 三、④⑤ 分种落词（草案，待你终审）

同一条带，④=发现（问题），⑤=修正（动作），按 critic 违规类型选。你的"超时"那对已锁死，其余是我按你的思考腔草的，你改字：

| 违规类型 | ④ 发现 | ⑤ 修正 |
|---|---|---|
| 超时（已锁） | 方案超出时间限制 | 让我压缩一下时间 |
| 超预算 | 花费超了预算 | 让我换些实惠的 |
| 桌型/坐不下 | 有的店坐不下这么多人 | 让我挑能容下的 |
| 忌口冲突 | 有一站不合忌口 | 让我避开这个 |
| 太远 | 有的点离得太远 | 让我找近一点的 |
| 营业时间 | 有的店那个点没开门 | 让我调下时段 |
| 兜底（未列类型） | 方案有个地方不太对 | 让我调整一下 |

> 注：后端违规 message 原文（"行程总时长 308 分钟…超出…"）太长太技术，**不进带**——带只按违规码取上面这些短句；原文只在展开细看时（若要）出现。

## 四、①③ 的 LLM 风格红线（你定，写进 prompt）

这两拍 LLM 每次现生成，你没法逐字手写，改为**定它的风格红线**：

**① 理解（intent 新字段 `understanding`）**
```
第一人称，像在心里过一遍：先点用户说的关键词，再说你据此判断了什么。
- 句式："用户……，我理解成……"
- 必须暴露一次推断（别太贵→预算敏感、4人→热闹），不是复述原话
- 一句、≤40 字、自然口语、不分点
- 禁词：为您/精心/智能/贴心/一站式/量身
```

**③ 规划（蓝图新字段 `plan_reason`）**
```
第一人称，说你这样安排的一个真实理由，扣住这一局的人和场景。
- 句式："用户……，所以先……"
- 理由要真（同伴/时间/氛围驱动），不是套话
- 一句、≤30 字、自然口语
- 禁词同上
```

## 五、四种收尾（诚实分档，别硬编高潮）

- **有自愈**（典型）：完整七拍，高潮 ④⑤⑥ 在；
- **一次过**（llm_first 无 critic）：无自愈戏，跳过 ④⑤⑥ → ①②③⑦，⑦ 仍"规划成功"，不吹改稿；
- **反馈改方案轮**：第二段小弧「按你说的'太远'重排 → 拉近了 → 规划成功」；
- **失败保留**：「试了几版都排不下，先保留上一版」——诚实不装。

## 六、header 标签（你问的，候选待你定）

header 是**这条带的常驻标签**（告诉评委"这是什么、可以点开"），不是结果句。**已定**：
- 规划中：**AI 幕后**（+脉冲点，表示正在思考）
- 出稿后：**AI 幕后**（静态）

结果（改了几版）由带里最后一拍 ⑦"规划成功"+ 冻结窗口里的 ④⑤⑥ 承担，header 保持干净，不塞"已重排两次"。

## 七、形态与动效（恒定高度，跳动不可能发生）

**恒定 3 行高**（细头 header + 3 行窗 ≈ 110px），所有状态高度一动不动 → 下方时间轴/地图永不跳。

```
┌───────────────────────────┐ ← header（常驻标签，规划中带脉冲）
├───────────────────────────┤
│ ⑤ 让我压缩一下时间         │  ← 3 行窗（恒定）
│ ⑥ 还是不行，换成算法引擎   │     · 自动模式：传送带渐入渐出
│ ⑦ 规划成功                 │     · 手动模式：锁自动、上下滚全历史
└───────────────────────────┘
   顶/底各一道渐隐遮罩（mask-image 渐变）
```

- **两模式（非变高）**：自动（规划中传送带自动推进）/ 手动（点击或悬停→停自动，同窗上下滚全历史）。"收起/展开"是行为不是尺寸，所以不跳。
- **传送带机制**：底部锚定，新拍底部淡入，旧拍上移，到顶淡出；容量 3 拍；不足 3 拍时预留 3 行高不撑不跳。
- **节奏解耦**：事件到达不均匀，加队列 + 每拍最小驻留 **800ms**，按可读节奏匀放（评委读得完）。
- **悬停暂停**：鼠标悬停停传送，移开继续。
- **自愈重音**：④⑤⑥ 三拍加一点**琥珀色**（全场卖点，值得被一眼认出），其余中性。
- **减动效降级**（prefers-reduced-motion）：关传送带，静态显示最新拍，瞬切。
- **时态**：规划中进行时（"让我压…"），冻结后完成态语气；冻结时窗口正好停在 ⑤⑥⑦（自愈高潮+定稿），评委最后一眼记住"它自己改了两版"。
- **动效参数**（只动 transform/opacity）：进场 opacity 0→1 + translateY(8px→0) ~400ms cubic-bezier(0.16,1,0.3,1)；推进整列 translateY(-1 行高) ~500ms；到顶淡出交给遮罩。

```
│ 思考   │ 时间轴+chip   │            │ ▸ AI 思考（信任带）   │ ← 恒定3行

└────────┴──────────────┘            │ 预告/动作             │
                                     └──────────────────────┘
```

## 修订（2026-07-06 真机反馈后 — 以下条款覆盖前文冲突处）

真机跑出四处问题，据此修订，落地以这一节为准：

1. **序号按实际步数递增，不用固定角色号**：拍在带里显示的圆圈序号 = 它在本轮实际序列里的位置（第 4 步就显 4），**不再**用 §二的 ①..⑦ 固定角色号（会出现"3 完直接跳 7"的观感）。①..⑦ 仅作设计文档里的角色代号，不是展示号。

2. **长出全部拍，废除 3 行窗 + 传送带滚动**（覆盖 §七"恒定 3 行高/两模式"）：真机证明——规划中方案还没渲染（下面是骨架屏），根本没有"下方内容"会被顶动；"恒定高度防跳"在防一个不存在的问题，代价却是把最该看到的①拍滚出视野。改为：**拍一条条淡入（保留 800ms 节奏 + 逐拍 fade-in），带高度随拍数自然长，①永远在、全弧可见**。出稿后静止全展示。自愈局的带会高一点（这是卖点证据，全摊在评委眼前反而好）。

3. **删「查看全部」+ 手动滚动模式**：全拍可见后无需滚动，`pinnedManual`/`hovering`/查看全部整块删除。

4. **单思考面**：删除 ChatDock 里 `thoughts.slice(-5)` 的原始 agent_thought 铺陈（那是带权重/冗长 rationale 的未加工重复面），对话框规划中只留一个"AI 正在思考…"脉冲；移动端同款删除。**AI 幕后（信任带）成为唯一思考面。** 筛掉权重等技术黑话是有意的；唯一真缺口——反馈上下文——由第 5 条补齐。

5. **refiner 加 understanding（反馈轮①拍上下文）**：与 intent 的 understanding 对称，给 refiner 输出 + prompt 加一个 `understanding` 字段，风格红线走反馈味——句式"用户说……，我理解成……"（如"用户说'太远了'，我理解成要拉近距离"）。这样反馈重规划轮，思考链①拍是"我在响应你这次的什么"，思考链自包含上下文。Optional 默认空串，不破既有。

6. （可选，未定）一次过时给③拍轻微视觉强调当"智力高潮"——本批暂不做，记档。

## 修订（2026-07-11 — 止损三修 + 五收据体系 + 定稿后折叠脊柱）

真机截图实锤：琥珀行渲染成「还是不行换引擎→让我压缩时间→方案超出限制→
还是不行换引擎」——因果倒序 + 一字不差重复，像系统抽风。本节收录三处止损修、
新增的五收据体系、以及定稿后的折叠脊柱形态，落地以本节为准（与前文冲突处
本节覆盖）。

### 一、止损三修（现有台面缺陷修复）

真机症状根因有三，均在 `lib/trust-belt.ts`：

1. **裁剪切断叙事对**：原 `buildTrustBeltBeats` 的预算裁剪对"拍的扁平数组"
   做 `slice`，预算不够整除时会把④⑤配对从中间切断（如只剩 `[fix, discover,
   fallback]`，丢了 fix 的配对 discover）——直接违反 §一.1"高潮必须两拍"。
   改为**以"轮"为最小裁剪单位**（`HealUnit`：一对 discover+fix，或一个独立的
   fallback 轮），裁剪时整轮保留或整轮丢弃，不再拆散配对。
2. **连续⑥不去重**：`FALLBACK_FIXED_TEXT` 曾是单一固定句，连续换引擎时一字
   不差重复，观感像系统抽风。改为**措辞变体池**（`FALLBACK_FIXED_TEXTS`，
   3 句，按 fallback 出现次序取用，用完停在最后一种）——仍是固定句家族
   （忠实不编：几句话说的是同一件"还是不行、再换个办法"的事，不是编一个
   没发生的新动作），保持第一人称思考腔、不带 §四禁词。
3. **跨引擎④⑤配对错挂**：`buildHealRoundsAndFallbacks` 原实现里，`current`
   （当前追踪的违规轮）会跨越 fallback 事件持续存活——若 fix_attempt 事件在
   fallback**之后**到达，会被错误地补给 fallback**之前**的旧违规轮（措辞用
   旧违规码的，seq 却顶着新引擎的时间戳），排序后修正句就跑到发现句前面。
   根治：fallback 处显式重置 `current`（fallback = 引擎切换 = 轮边界）。

三修均有 vitest 单测覆盖（复现真机症状的 criticReport fixture，断言修后
叙事序正确），见 `lib/trust-belt.test.ts` "止损三修（真机症状回归）" 一节。

### 二、五收据体系（琥珀专属权重申：收据全部中性墨色）

信任带的"忠实不编"卖点延伸出五处收据——每处收据都是**某一拍下挂的附件**，
不是新的一拍（同②拍检索收据芯片的既有纪律），全部 Lucide 单线图标、禁
emoji、**全程中性墨色**——**琥珀色仍是④⑤⑥自愈拍专属重音，收据不挪用**
（本节重申 §七"自愈重音"这条红线，五收据无一例外）。

| 收据 | 落位 | 图标 | 数据源 |
|---|---|---|---|
| 画像 | ①拍下 | `User`（**虚线边框**，与②拍芯片实线边框区分出处：实线=查到的，虚线=记得的） | `intent.field_provenance` 标为 `prior` 的受控词典字段（dietary_constraints/physical_constraints/experience_tags）——只显示**真被这句话的过滤消费**的字段，不是"画像被拉取了"就显示；后端 `_emit_handlers.py::_consumed_profile_fields`，挂在 `get_user_profile` 工具的 `tool_call_end.output.profile_fields` |
| 放宽重搜 | ②拍芯片行尾 | `TriangleAlert`（中性墨色文字承担诚实） | 已有的 `relaxed_tags`（本次丢弃的第一个 soft tag）+ `count`（放宽后的真实候选数），拼成「"安静"不够，放宽后补到 12 家」；前端 `buildRelaxedSearchNotice` |
| 质检 | ⑦拍下 | `ShieldCheck`「N 项体检 ✓」，v1 不可点开 | `agent.planning.critic.validate.REGISTRY` 现场计数（不是硬编码常量，REGISTRY 增删 check 自动跟着变），挂在 critic 通过分支的 `agent_thought.checks_run` |
| 记忆 | ⑦拍后独立行（晚到淡入） | `Bookmark`「记住了：…」 | 已有的 `MEMORY_PERSISTED` 事件（`/chat/confirm` 才推，到达天然晚于⑦拍）；`success=false` 不显示 |
| 换菜备选 | **不进信任带**——挂换菜结果叙事下方 | `ArrowLeftRight`「同类替补 N 家」 | 换菜成功那条 `AGENT_NARRATION` 新增可选 `swap_alternatives_count`（服务端在换菜发生的同一次请求里直接算好 `node_actions[swapped_to].alternatives` 的长度，不需要前端做"这个节点是不是刚被换过"的脆弱映射） |

修订4），这行的落点是换菜响应 UI（`ItineraryCard.tsx` 的 `NarrationBlock`
正下方），不是信任带的第八种收据。

### 三、定稿后折叠脊柱

**与修订2 的关系（必须先讲清楚，避免误读成"推翻修订2"）**：修订2 反对的是
**直播过程中**用户手动滚动导致①拍被顶出视野（"恒定高度防跳"在防一个不存在
的问题，代价是最该看到的①拍反而滚没了）——它保护的是**直播态**，主张"全部
拍逐个淡入、全展示、不滚动"。折叠脊柱发生在**定稿之后**（⑦拍 reveal 完成
+ 停顿之后），保护的是**交付物台面主权**——评委的注意力这时候该完全给
方案卡，不该被一条已经讲完的思考带占着地方；证据没有消失，只是换成脊柱
形态**常驻**在原来的位置（对象恒存：评委刚看过的东西换形态，没有被隐藏或
删除）。两次修订管的是同一条带的两个不同阶段，不冲突。

**状态机**（三态：`live` / `spine` / `expanded`，实现细节含一个内部过渡态
`collapsing` 供动效播放窗口，不是第四个可长期停留的状态）：

```
live（逐拍淡入全展示，修订2 既有行为原样保留）
  │ ⑦拍 reveal 完成 + 停约 1 秒 + 非 give_up/降级收尾 + 当前未悬停
  ▼
spine（折叠：一行横向图标链，每节=原拍图标缩影+关键数字，琥珀节保持琥珀）
  │ 点击脊柱任意处                    │ 再次点击
  ▼                                   │
expanded（完整带，全部拍一次性可见，不重播逐拍节奏）──────┘
```

- **收拢动效**：300-400ms，只动 `transform`/`opacity`（不animate 高度本身，
  避免"从 auto 到固定值"的技术难题，用 opacity+scale 进场代替真实高度
  收缩的观感）；`prefers-reduced-motion` 时瞬切到 spine，不播过渡。
- **关键数字来源**：严格限定为"已经算好的真实数据"，没有可靠来源的拍种
  （①理解、④⑤发现修正、⑥换引擎）**不编数字**，只显示图标——忠实不编在
  脊柱这一层同样成立。②检索=本轮真实召回总数；③规划=最终方案活动节点数；
  ⑦定稿=质检收据数字（同上方"质检"收据同一数据源）。见
  `lib/trust-belt.ts::buildSpineNodes`。
- **例外（不自动折叠）**：§五"失败保留"/`give_up` 降级收尾——失败保留的
  诚实证据该一直摊在台面上，不该被"看起来完成了"的折叠态误导评委。
- **悬停/滚动推迟折叠**：用户鼠标停留在带上（含收拢过渡进行中途悬停会中止
  收拢、退回 live）时不折叠，不能在用户还在看的时候把内容收走。
- **记忆收据晚到**：若 `MEMORY_PERSISTED` 在已经处于 spine 态之后才到达
  （`/chat/confirm` 时机通常如此），追加落在脊柱尾部淡入（`Bookmark` 图标
  节点），不需要先展开完整带才能看见。
- **新一轮规划开始**：store 重置（itinerary 从"已就绪"变回"未就绪"）→ 状态
  机回 `live`，新一轮从头逐拍演出——同一个 `<TrustBelt/>` 实例跨轮复用，不
  依赖组件重新挂载。
