# 信任带设计终稿（AI 思考流）

> 定位：把原来散在左栏的三个技术面板（ToolTracePanel「Agent 思考链路」/ ThoughtPanel「Agent 在想什么」/ DecisionTraceCard「决策链路」）合成**一条**，改成"评委看着 AI 一路想下来"的第一人称思考流（DeepSeek 推理腔）。核心卖点：让评委*感受到*"它真听懂了人话、真在解题、还会自己揪错改掉"。
> 状态：设计定稿，待两处你终审（④⑤ 分种落词、header 标签）后落地。不动代码。

---

## 一、叙事逻辑（评委视角的三幕弧）

评委盯着看的 10-20 秒里，要读到一个故事：**听懂 → 干活 → 自省自愈 → 定稿**。高潮是"自省自愈"，它是和"套壳 ChatGPT"的分水岭。

关键原则（我这次想透的增量）：
1. **高潮必须两拍**——先亮问题（④），再亮修正（⑤）。只显示"已修正"评委无感；先"超时了"再"压一压"，评委才**看见它揪自己的错**。
2. **信息流要剪辑**——后端事件（15 条思考 + 3 次返工 + critic + fallback）不能直灌。选（15 条思考压成 3 拍）、并（3 次同样返工合成"发现→压→换引擎"的递进）、限（总量 ~7 拍）。
3. **忠实不编**（承接"叙事不编因"纪律）——每拍的具体词从**真实事件数据**生成，①③ 是真 LLM 输出，④⑤ 由真实违规类型驱动，不硬编一个没发生的故事。

## 二、七拍映射全表（事件 → 拍 → 来源 → 落词）

整条带统一**第一人称思考腔**（"让我…""用户…我理解成…"）。

| 拍 | 触发事件 | 类别 | 落词 |
|---|---|---|---|
| ① 理解 | `intent_parsed`（新 `understanding` 字段） | **LLM 现生成** | 例：*用户想周五晚和室友唱K，还说别太贵——我理解成想热闹、但钱别花太多*（风格红线见 §四） |
| ② 检索 | `tool_call`（search_pois/restaurants） | **固定句** | **让我先查询附近的店铺和时间** |
| ③ 规划 | 蓝图生成（新 `plan_reason` 短字段） | **LLM 现生成** | 例：*用户同行年轻人多，所以先用 KTV 带动气氛*（因人而异，LLM 每次真给；风格红线见 §四） |
| ④ 发现问题 | `critic_violations`（按违规码） | **分种模板** | 见 §三（超时那种＝**方案超出时间限制**） |
| ⑤ 修正 | `critic_fix_attempt` / backprompt | **分种模板** | 见 §三（超时那种＝**让我压缩一下时间**） |
| ⑥ 换引擎 | `plan_fallback`（→ ILS） | **固定句** | **还是不行，换成算法引擎** |
| ⑦ 定稿 | `done` / `itinerary_ready` | **固定句** | **规划成功** |

## 三、④⑤ 分种落词（草案，待你终审）

同一条带，④=发现（问题），⑤=修正（动作），按 critic 违规类型选。你的"超时"那对已锁死，其余是我按你的思考腔草的，你改字：

| 违规类型 | ④ 发现 | ⑤ 修正 |
|---|---|---|
| 超时（已锁） | 方案超出时间限制 | 让我压缩一下时间 |
| 超预算 | 花费超了预算 | 让我换些实惠的 |
| 桌型/坐不下 | 有的店坐不下这么多人 | 让我挑能容下的 |
| 忌口冲突 | 有一站不合忌口 | 让我避开这个 |
| 太远 | 有的点离得太远 | 让我找近一点的 |
| 营业时间 | 有的店那个点没开门 | 让我调下时段 |
| 兜底（未列类型） | 方案有个地方不太对 | 让我调整一下 |

> 注：后端违规 message 原文（"行程总时长 308 分钟…超出…"）太长太技术，**不进带**——带只按违规码取上面这些短句；原文只在展开细看时（若要）出现。

## 四、①③ 的 LLM 风格红线（你定，写进 prompt）

这两拍 LLM 每次现生成，你没法逐字手写，改为**定它的风格红线**：

**① 理解（intent 新字段 `understanding`）**
```
第一人称，像在心里过一遍：先点用户说的关键词，再说你据此判断了什么。
- 句式："用户……，我理解成……"
- 必须暴露一次推断（别太贵→预算敏感、4人→热闹），不是复述原话
- 一句、≤40 字、自然口语、不分点
- 禁词：为您/精心/智能/贴心/一站式/量身
```

**③ 规划（蓝图新字段 `plan_reason`）**
```
第一人称，说你这样安排的一个真实理由，扣住这一局的人和场景。
- 句式："用户……，所以先……"
- 理由要真（同伴/时间/氛围驱动），不是套话
- 一句、≤30 字、自然口语
- 禁词同上
```

## 五、四种收尾（诚实分档，别硬编高潮）

- **有自愈**（典型）：完整七拍，高潮 ④⑤⑥ 在；
- **一次过**（llm_first 无 critic）：无自愈戏，跳过 ④⑤⑥ → ①②③⑦，⑦ 仍"规划成功"，不吹改稿；
- **反馈改方案轮**：第二段小弧「按你说的'太远'重排 → 拉近了 → 规划成功」；
- **失败保留**：「试了几版都排不下，先保留上一版」——诚实不装。

## 六、header 标签（你问的，候选待你定）

header 是**这条带的常驻标签**（告诉评委"这是什么、可以点开"），不是结果句。**已定**：
- 规划中：**AI 幕后**（+脉冲点，表示正在思考）
- 出稿后：**AI 幕后**（静态）

结果（改了几版）由带里最后一拍 ⑦"规划成功"+ 冻结窗口里的 ④⑤⑥ 承担，header 保持干净，不塞"已重排两次"。

## 七、形态与动效（恒定高度，跳动不可能发生）

**恒定 3 行高**（细头 header + 3 行窗 ≈ 110px），所有状态高度一动不动 → 下方时间轴/地图永不跳。

```
┌───────────────────────────┐ ← header（常驻标签，规划中带脉冲）
├───────────────────────────┤
│ ⑤ 让我压缩一下时间         │  ← 3 行窗（恒定）
│ ⑥ 还是不行，换成算法引擎   │     · 自动模式：传送带渐入渐出
│ ⑦ 规划成功                 │     · 手动模式：锁自动、上下滚全历史
└───────────────────────────┘
   顶/底各一道渐隐遮罩（mask-image 渐变）
```

- **两模式（非变高）**：自动（规划中传送带自动推进）/ 手动（点击或悬停→停自动，同窗上下滚全历史）。"收起/展开"是行为不是尺寸，所以不跳。
- **传送带机制**：底部锚定，新拍底部淡入，旧拍上移，到顶淡出；容量 3 拍；不足 3 拍时预留 3 行高不撑不跳。
- **节奏解耦**：事件到达不均匀，加队列 + 每拍最小驻留 **800ms**，按可读节奏匀放（评委读得完）。
- **悬停暂停**：鼠标悬停停传送，移开继续。
- **自愈重音**：④⑤⑥ 三拍加一点**琥珀色**（全场卖点，值得被一眼认出），其余中性。
- **减动效降级**（prefers-reduced-motion）：关传送带，静态显示最新拍，瞬切。
- **时态**：规划中进行时（"让我压…"），冻结后完成态语气；冻结时窗口正好停在 ⑤⑥⑦（自愈高潮+定稿），评委最后一眼记住"它自己改了两版"。
- **动效参数**（只动 transform/opacity）：进场 opacity 0→1 + translateY(8px→0) ~400ms cubic-bezier(0.16,1,0.3,1)；推进整列 translateY(-1 行高) ~500ms；到顶淡出交给遮罩。

## 八、位置与布局（侧栏取消、方案全宽）

信任带插在**叙事和时间轴之间**（pitch → 取信 → 细节的说服顺序）；**绝不进时间轴和地图之间**（那会把同一方案的两个视图劈开）。左侧栏取消，方案独占全宽；进房间时顶部长出约束栏。

```
改前（两栏）                          改后（全宽单脊柱）
┌────────┬──────────────┐            ┌──────────────────────┐
│ 工具   │ 叙事          │            │ 叙事                  │
│ 思考   │ 时间轴+chip   │            │ ▸ AI 思考（信任带）   │ ← 恒定3行
│ 决策   │ 地图          │            │ 时间轴+chip           │
│ 约束   │ 预告/动作     │            │ 地图                  │
└────────┴──────────────┘            │ 预告/动作             │
                                     └──────────────────────┘
```

## 修订（2026-07-06 真机反馈后 — 以下条款覆盖前文冲突处）

真机跑出四处问题，据此修订，落地以这一节为准：

1. **序号按实际步数递增，不用固定角色号**：拍在带里显示的圆圈序号 = 它在本轮实际序列里的位置（第 4 步就显 4），**不再**用 §二的 ①..⑦ 固定角色号（会出现"3 完直接跳 7"的观感）。①..⑦ 仅作设计文档里的角色代号，不是展示号。

2. **长出全部拍，废除 3 行窗 + 传送带滚动**（覆盖 §七"恒定 3 行高/两模式"）：真机证明——规划中方案还没渲染（下面是骨架屏），根本没有"下方内容"会被顶动；"恒定高度防跳"在防一个不存在的问题，代价却是把最该看到的①拍滚出视野。改为：**拍一条条淡入（保留 800ms 节奏 + 逐拍 fade-in），带高度随拍数自然长，①永远在、全弧可见**。出稿后静止全展示。自愈局的带会高一点（这是卖点证据，全摊在评委眼前反而好）。

3. **删「查看全部」+ 手动滚动模式**：全拍可见后无需滚动，`pinnedManual`/`hovering`/查看全部整块删除。

4. **单思考面**：删除 ChatDock 里 `thoughts.slice(-5)` 的原始 agent_thought 铺陈（那是带权重/冗长 rationale 的未加工重复面），对话框规划中只留一个"AI 正在思考…"脉冲；移动端同款删除。**AI 幕后（信任带）成为唯一思考面。** 筛掉权重等技术黑话是有意的；唯一真缺口——反馈上下文——由第 5 条补齐。

5. **refiner 加 understanding（反馈轮①拍上下文）**：与 intent 的 understanding 对称，给 refiner 输出 + prompt 加一个 `understanding` 字段，风格红线走反馈味——句式"用户说……，我理解成……"（如"用户说'太远了'，我理解成要拉近距离"）。这样反馈重规划轮，思考链①拍是"我在响应你这次的什么"，思考链自包含上下文。Optional 默认空串，不破既有。

6. （可选，未定）一次过时给③拍轻微视觉强调当"智力高潮"——本批暂不做，记档。

## 修订（2026-07-10 — ②拍检索收据芯片）

**动机**：②拍是固定句"让我先查询附近的店铺和时间"，评委听不出"它真查到了什么"——
这条带的卖点之一是"忠实不编"，②拍却是全带唯一一处纯固定句、不挂真实数据的拍。
同时这也是刚修完的 anchor-escape bug（显式点名某类场所必须真召回）最需要一块
"舞台可见证据"的地方：芯片直接证明"用户点名的东西，系统真的搜到了"。

**规格摘要**（不改变七拍剪辑纪律——芯片是②拍正文下挂的附件，不是新的一拍）：
- 数据：餐厅 + POI 两类各取召回评分 top-3，餐厅组在前、POI 组在后；尾部
  "+N" 徽章（N=两类总召回数−已展示数，≤0 不显示）；两类召回都为 0 时整行
  芯片不出现，②拍保持纯文字。
- 内容：图标（Lucide 单线，餐厅=UtensilsCrossed / POI=MapPin，禁 emoji）+
  名字（截断；展示名剥尾部分店括号后缀——「绿茶餐厅(凯德MALL店)」显示为
  「绿茶餐厅」，全角/半角都剥、只剥尾部、剥空回退原名，见 `chipDisplayName`，
  拍板 2026-07-10）+ 评分（一位小数）。全程中性墨色——琥珀色仍是④⑤⑥自愈拍
  专属重音，芯片不挪用。不可点击（v1 拍板）。
- 数据链路：fan-out 搜索 worker 的 `tool_call_end.output` 新增可选 `preview`
  字段（`{kind, name, rating}[]`，评分 top-3，"无召回不加字段"），后端见
  `agent/graph/_emit_handlers.py::_top_rated_preview`；前端剪辑纯函数
  `lib/trust-belt.ts::buildSearchPreviewChips`，渲染在
  `components/TrustBelt.tsx::SearchPreviewChipRow`。
- 进场动效：②拍文字进场后 300ms 开始，芯片从左到右每 70ms 一个错峰
  （fade + 上浮4px + scale 0.96→1，同 `trustBeltEnter` 的缓动曲线），"+N"
  最后进场；挂载时方案已就绪、或 `prefers-reduced-motion` 时瞬显无动画
  （与既有两条行为对齐，见 `TrustBelt.tsx` 组件内注释）。
