## 二、关键决策

- **D1 · 叙事五段结构（想法阶段，未全实现）**：痛点 → 成品方案 → **系统概览地图** → 局部方法深讲 → 商业价值。
  - prior art：先行组织者 / 概览优先（Shneiderman）/ 预训练原则——先给地图，局部讲解才落到实处。
  - 顺手焊掉旧裂缝：旧版喊"走完整个系统"但只深讲了路由；新结构里**地图=全系统(广度)、深讲=挑最硬模块看透(深度)**。
  - 状态：概览地图**已建**（形态选「流水线全景」）；**旁白收口**（`s_type`/intro 不再喊"走完整个系统"、降格为"理解·路由特写"）+ **商业价值结尾 `s_value`** = 已出草案、**PARKED**，待完整场景 + 逻辑想透后再实现。
- **D2 · #28 性能（已完成）**：measure-first 实测 → box-shadow"非合成"flag 的主源是**静态阴影 + transform 动画**（拆全片阴影会毁美团风质感，不值）。只把 5 处**反复脉冲辉光**（建关 hook / runthrough g0·g1·g2 / planning）迁成 `.glow-layer` 子层的 opacity（走合成器）；token 折叠、act 失败红等**一次性** box-shadow 不迁（负 ROI）。影响始终可忽略。
- **D3 · 工作方式**：设计/叙事/功能/重构类 → 先 solve-it-right 想透 + 呈现方案，**用户明说"实现/改代码"才动码**（已配 think-first 钩子自动提醒）。

## 三、想法 / 待办（parked / optional）

- 叙事改版落地：旁白收口 + `s_value` 商业价值结尾（需先完整场景设计 + 逻辑想透）。商业价值草稿 3 点：①从"逛"到"做完"成交闭环 ②大多数请求连大模型都不进（便宜可规模化）③半日局高频刚需、换画像即新场景。
- 选做精修：连线行进光点 / 翻面高光扫过 / token 拖影（未开始）。
- reduced-motion 低动效开关（a11y · WCAG 2.3.3，满屏 blur+粒子对前庭敏感者；可选保险）。
- timecut 导出备份视频（无网/发给跑不了 HTML 的人时；可选）。

## 四、工具 / 自测要点

- **chrome-devtools MCP**：逐拍 next() 自测、量 getBoundingClientRect、`gsap.globalTimeline.timeScale()` 慢放抓粒子中途帧、`performance_start_trace` 查掉帧。
- **gsap-skills**（全局，纯知识）、**pixelmatch + pngjs**（`路演PPT/_vrt`，防回归）。
- 驱动坑：`gotoScene(si)` 瞬时重放会留 caption 残影 → 判状态以 DOM 计算值为准；**续播片段（addPause 之后）首元素必须是真 tween，不能是落在 pause 边界的零时长回调**，否则 play() 续播会跳过紧贴其上的 tween。
