## 一、写作准则（每张卡都钉死）

1. **零背景假设**：当读者完全不懂这个项目来写（评委没有内部模型，作者自己隔几天也没有）。
2. **首次出现即解释**：任何项目名词（6 种义务/一脑三壳/澄清地板/critic/ILS/蓝图…）第一次出现，**先用人话说清"是什么+干什么+在哪一环"，再（可选）给术语名**。反例：直接甩"澄清地板"。正例："大模型拿不准这句话要干什么时，系统不瞎猜、退回去先反问一句——这个'拿不准就追问'的底线，我们叫澄清地板。"
3. **人话先于术语**：能用人话就不甩黑话；非用不可，先人话后术语。
4. **逻辑承接**：第二句接住第一句往前走（起承转），不是换个说法重说、不是并列堆砌。
5. **意群分段**：一段一个意思，新意群另起一段；一张卡的答案不堆成一坨。
6. **自包含**：每张卡答案独立成立，不依赖读者记得别的卡。
7. **一分钟版先答结论**：正文开头直接回应问题本身，再展开。

## 二、三支柱 + 万能锚点（message house，答案的脊柱）

**三支柱**（任何答案最后收到其一）：
- ①**体验闭环**——一句话到可下单，反馈/换菜/组局协商全链路真实可用；
- ②**架构判断力**——全后端约 120 个决策点逐一划界（哪些归大模型、哪些归代码），每次划界有实测数据；
- ③**工程护栏**——大量单测 + 端到端探针 + 15 份架构决策记录，每个修复先实锤根因再动手。

**万能锚点句**："在该自由的地方自由（语义判断、长尾问答），在该确定的地方确定（约束求解、审计、安全）——每一次划界，都有实测数据。"

**万能收束**："理解归模型，计算归代码，中间是类型化契约。我们不是不知道 autonomous agent，是查过它在这类任务上 0.6% 的通过率之后，把自由用在了它该在的地方。"

## 四、术语人话对照表（反黑话核心 · 每个术语的标准人话解释）

**代理引用任一术语，第一次必须用下面这一句人话解释它，再用术语名。** 不许绕过。

| 术语 | 一句人话（首次出现时这样解释） |
|---|---|
| **一脑三壳** | 路由层的结构：用户每说一句话，先过三层"壳"（快速、确定性的规则关卡），挡不掉的才交给"脑子"（一次大模型调用）判断该干什么。 |
| **壳1（注入检测）** | 第一层壳，安全关卡——用确定性规则拦恶意指令（"忽略之前设定"这类），这活不能交给可能正被攻击劫持的大模型。 |
| **壳2（字面短路）** | 第二层壳——用户点的按钮文本是系统自己生成的，百分百确定，直接秒回，不浪费一次大模型调用。 |
| **壳3（保守地板/澄清地板）** | 第三层壳——大模型拿不准这句话要干什么（置信度低于阈值）时，系统不瞎猜、退回去先反问一句。这个"拿不准就追问、绝不瞎做"的底线。 |
| **脑子（路由大模型）** | 三层壳挡不掉的话，交给一次大模型调用，判断这句话要系统干什么。 |
| **6 种义务（响应义务）** | 系统对每句话判断它属于哪一类"该干什么"——规划（出新方案）/反馈（改现有方案）/确认（拍板下单）/澄清（信息不够，反问）/问答（问方案里的事）/闲聊。这 6 类是系统能做的回应的闭集。 |
| **意图抽取** | 大模型把用户的话变成结构化的需求字段（时间/同伴/预算/忌口/活动偏好…），供后面规划用。 |
| **出处四值** | 意图每个字段记着它从哪来——用户明说的/推断的/档案里的/系统默认的；用来分硬软约束、诚实告知。 |
| **蓝图** | 规划时大模型只做"选择"——去哪、做什么、待多久，不碰时间计算；这个选择骨架叫蓝图，时间轴/通勤/预算由代码算。 |
| **critic（校验器）** | 方案排好后一道确定性检查关卡，查 16 项硬指标（时间冲突/预算超支/忌口/桌型/营业时间…）；它是代码不是大模型，有标准答案。 |
| **ILS（算法引擎）** | 一个求解算法（迭代局部搜索）；大模型排的方案校验不过又改不好时，交给它重排——它擅长解"带约束的路线安排"这类组合优化题。 |
| **LLM-Modulo（求解器在环）** | 架构范式：大模型负责提案和品味，确定性求解器负责保证可行；业界应对"大模型做规划通过率低"的成熟解法。 |
| **诉求台账** | 记着用户提过的每个诉求（谁要的什么），跨轮不丢。 |
| **出口审计** | 方案定稿时逐字段比对"用户要的满足了没"，没满足的诚实告知。 |
| **换菜（局部重解）** | 用户对某一站不满意，点按钮换掉这一个节点，不推翻整个方案。 |
| **信任带（AI 思考流）** | 界面上一条第一人称的思考流，让评委看着 AI 一路想下来（听懂→查店→排店→自己发现问题→改→定稿），展示"它会自省自愈"。 |
| **约束池水位线** | 多人房间里用户反复改主意时，只把"上次重排之后新增的话"当最新指令，旧话不再反复重播——保证"用户最新的话最高优先级"。 |
| **词表精度契约** | 留在规则层的关键词，每条必须单独接近百分百精度，做不到就剪、召回交给大模型；因为规则层命中即拍板、无兜底。 |

（探索中若遇到表里没有的术语，按同样规格自己写一句人话解释，并在交付里回报，我并进表。）
