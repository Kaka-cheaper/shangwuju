# 卡片主角化 + 事实面板设计终稿

> 一次解决两件互相纠缠的事:④「降黄后方案没重点了」+ ③「节点卡右侧空白填真实数据详情」。
> 状态:方向已拍板——暖金顶冠(持续强调)+ 到场重音(所有方案都加)+ 安静右栏事实面板(数据来源已确认真实数据 only)。
> 落地边界:前端(ItineraryCard.tsx Web + MobileHomeView 移动端镜像)。数据管道(node_detail)后端已全链路就绪(/chat/turn + 换菜单人/房间 + 房间快照)。

---

## 一、诊断:降黄没错,是矫枉过正(概念完整性根)

- 原病:70% 全黄 → 强调预算超支 7 倍 → 没重点。**降黄对症,方向对。**
- 新病:降黄把**信号连噪音一起降了**。现在聊天/卡片/时间轴全中性灰,**最该被强调的"刚形成的那份方案"和一个工具按钮长得一样平**——焦点不是被淹,是被拉平。
- 命名:**视觉层级 / 焦点**——60-30-10 的"10%"强调必须落在全屏唯一主角身上;降黄把主角的 10% 也擦了。**峰终 / 到场时刻**——方案"形成"是体验高潮,该有视觉重音。
- 修法铁律:**不是把黄加回去铺满(退回原罪),是把强调重新聚到"方案"这一个主角上。** 信噪比:噪音已降,现在只给主角把信号提上去。

## 二、④ 方案卡主角化:暖金顶冠 + 到场重音

### 2.1 暖金顶冠(持续强调)
- 现状:卡头已有「📷 今日行程安排」焦糖字(#8f4b24)+ 暖色胶带装饰 + 大标题 summary + 时长 pill。**warmth 已经在,只是太弱、没成为"冠"。**
- 改法:把卡头那一行升成一道**暖金冠带**——标题行背后一层克制的暖金渐变(accent 深金域,低透明、柔和,非亮黄实底),托住「今日行程安排」+ 时长 pill。卡一眼宣告「我是交付物」,靠的是这道冠、不是铺黄。
- 大标题 summary 保持中性深墨(text-ink-900),冠带只在最顶那一行——**主角的"身份牌",不是整卡染黄**。

### 2.2 全卡金预算(严格三处)
金色在方案卡上**只出现三处**,其余全中性:
1. **暖金顶冠**(交付物身份);
2. **确认并预约 CTA**(brand-500 亮黄实底,全站唯一——已是,不动);
3. **每个节点的评分星 ⭐**(事实面板里唯一暖色触点,见 §三)。
自检:一屏方案卡上数暖色元素 = 顶冠 1 + CTA 1 + N 个评分星(小、点状)——聚焦,不铺。

### 2.3 到场重音(所有方案落定都加,最贴"形成方案那感觉")
方案从无到有(itinerary null→非 null)那一刻:
- 卡顶暖金光带**扫过一次**(复用现有 shimmer-bar 机制,颜色走暖金);
- 卡片 spotlight-once 脉冲(已有,保留);
- 信任带「定稿」拍点亮暖金(已有 accent,确保这一拍是暖金重音);
- ~2 秒后归于平静(不持续占用视觉,靠 §2.1 顶冠维持静止态的主角身份)。
- reduced-motion:跳过扫光,直接静止态顶冠 + 定稿拍。

## 三、③ 右栏事实面板(安静 ⊂ 主角强调)

### 3.1 为什么安静是设计而非将就
正因为方案卡整体要当主角(§二),**事实面板一响就和主角抢焦点**。所以面板克制**不是省事,是服务层级**——③的安静正好成全④的强调,两者一套连贯设计。

### 3.2 两栏节点卡
- 左栏(现状不动):玻璃标签 + 店名 + 选店理由 + 操作行(备选/微调/投票);
- 右栏(新,~140px):紧凑事实面板,数据来自 `node_detail`(后端已下发,四条路径全覆盖)。

### 3.3 字段(curate,不堆全;餐厅/POI 各显各的)
| 展示位 | 餐厅 | POI |
|---|---|---|
| 评分 | ⭐`rating`(**唯一暖色触点**,深金星) | 同 |
| 价钱 | `price_text`「¥90/人」 | 「¥80–120」/「免费」 |
| 距离 | 🚶`distance_km`「4.5km」 | 同 |
| 可订/余位 | `availability_text`「可订18:00」/「需排队」 | 「余45」/「约满」 |
| 标签×1-2 | 桌型「大圆桌/有包间」+ 1 氛围 | 适龄「适合5-10岁」+ 1 |
| 营业 | `open_until_text`「营业至21:30」(可选) | 同 |

### 3.4 克制参数
- 事实文字:幽灰微字(text-ink-500/400,~11-12px);
- 评分星:深金 accent(唯一暖色);tag:极小中性 chip(bg-black/[0.03] text-ink-600);
- 字段缺失(后端 `exclude_none` 已省略)→ 前端该位不渲染,不留空占位;
- 面板与左栏之间一道极淡竖分隔(或留白),不加重边框。

### 3.5 两个小修(一起做)
- **店名不再硬截**:右栏兜住了卡宽,店名可换行/宽松截断(现状 `truncate` 单行 → 放宽);
- **备选按钮去品类后缀**:「老王烧烤摊·路边烧烤」→「老王烧烤摊」(只留店名,品类后缀丢了辨识度那半截不如不显);仍留 title 全名兜底。

## 四、三层自检

**功能内部**
- 字段 curate 不堆全(信息气味);POI/餐厅字段集不同,分支渲染;
- 可订/余位只报真值(后端诚实红线已落:真槽/需排队/约满/省略),前端**照实显、不美化**;
- 缺字段优雅隐藏(后端 exclude_none + 前端条件渲染)。

**功能间联动**
- 事实面板安静 ⊂ 方案卡强调——③克制服务④层级,不互抢;
- `node_detail` 四路径全覆盖(换菜后面板不消失,②已修);
- 暖金顶冠 vs 确认 CTA:冠=暖金柔和(身份)、CTA=亮黄实底(唯一主行动),**两种金不同角色不打架**;评分星金 vs 顶冠金都"在主角身上",同族自洽;
- 移动端 MobileHomeView 镜像:顶冠 + 事实行(移动端窄,右栏改为店名下一行事实 chips,不强上两栏)。

**概念完整性**
- 金聚到方案这一个主角(顶冠+CTA+评分星)= 焦点系统恢复;其余全静 = 降黄成果保住;
- **事实归数据、强调归层级(不靠铺色)**——和 ADR-0015「计算归数据」+ 全站克制同一套思想;
- 到场重音让"形成方案"成为可感的高潮,呼应峰终。
