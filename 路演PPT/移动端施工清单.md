## 强烈建议:先抽 3 个共享 hook,再补 UI(治本,防以后再漂移)

审计发现漂移根源是"调用逻辑粘在 UI 组件里"。补 UI 前先把这三段抽成与 UI 无关的共享函数,移动端直接调,从根上杜绝"忘记分流":

1. **`useCollabDispatch()` / `sendUserInput(text)`**:collabMode 分流(WS constraint vs HTTP sendMessage)——现已在 ChatDock+ChitchatBubble 逐字重复两遍,A5 会是第三遍;抽出来三处共用。
2. **`useConfirmAction()`** 返回 `{canConfirm, handleConfirm, confirmLabel}`:含房主守卫——治 A6。
3. **`useBootstrapPlannerMode()`**:把 cookie/health 校准从徽章 useEffect 挪出——治 A9。

另有一个"共享代码里的洼地"可顺手治:`event-handlers.ts:361-365` 的 stream_error 只 set 不 toast——在这里补 pushToast,**两端共用同一 handleEvent,修一处两端都受益**(A8 的根治版)。
