# 晌午局 · 路演设计宪法 v1

> 一页纸。每一幕落地前都拿它过一遍;过不了就回炉。

## 0 · 北极星
这套作品**没有一块是多余的——每一层都是被一个我们真撞过的问题逼出来的**。
演示要让评委看见这个"被逼出来"的过程,而不是看见一张功能清单。

## 1 · 叙事骨架:倒叙先亮成品 → 受逼成长
- **先 wow**:开场倒叙,甩出成品的魔法瞬间(一句话 → 一份能成行的方案),让评委先"哇"、先信"真能跑"。
- **再 earn**:之后每一幕用「受逼成长」讲——
  1. 直觉上的**最简主线**:用最少的零件先跑通核心;
  2. 抛出最简版**扛不住的一个具体问题**;
  3. **只加解决它所需的最小一环**;
  4. 再抛、再加。
- 每一环都**挂着它专门解决的那个失败案例**。

## 2 · 三条铁律(不可违背)
1. **wow 与深度,都要,不二选一。** demo 证明"真能跑"(信任),受逼成长证明"不简单"(深度)。删掉任何一个都掉分。
2. **深度预算:只在 1–2 个差异化最强处递归到底。** 其余环节讲一层就够,甚至一句话带过。均匀递归 = 节奏垮 + 评委累。
3. **问题是"领域本来就难",不是"我们犯的错"。** 同一事实,讲成"朴素做法必踩的坑 / 大多数人没想到的失败模式" = 洞察(加分);讲成"我们有个 bug 后来修了" = 不靠谱(减分)。

## 3 · 每个环节的"出生证"(答不全就别上台)
任何一环上台前,必须一口气答出五问:
1. 它的**最简主线**是什么?(没有它,怎么个最简法)
2. 最简版**扛不住什么**?(一个具体、可演的失败:某句输入 / 某个数字 / 某种攻击)
3. 为解决它,**加了哪最小的一环**?(不多加)
4. 加了之后**凭什么就扛住了**?
5. **优势**在哪?(比朴素做法、比别人好在哪)

## 4 · 认知节奏
- **一拍一个意思**:每次推进只抛一个点,塞不进第二个。
- **缺口先于揭晓**:先抛"这会出什么事?"的真问题(带 ?),再给做法。
- **先给骨架**:每个大环节开头先用一句话给"这一段要去哪"的路线图,别让评委在细节里迷路。

## 5 · 文案(去 AI 味)
- **短句、具体、用证据**:能给场景/数字就别给形容词("周日两点开到第 8 个 App"胜过"决策困难")。
- **全程中文人话**:不分 L 几、不甩 `planning_fast_path` 这种代号;术语先翻成功能名。
- **对仗/排比全篇限一处**:只留给主张句「LLM 决主观,算法决客观」。其余老实把话讲清,不堆"赋能/闭环/纵深"。

## 6 · 视觉与交互
- **美团黄白黑**:跟产品前端同皮(白底 + 美团黄 + 黑字 + 琥珀强调)。
- **一镜到底**:同一个对象连续变形,不是一堆独立幻灯片;**动效是搬运信息的镜头,不是装饰**。
- **克制**:不满屏乱飞;留白、慢镜、有重量。
- **每按一步**:节奏由讲解者掌控,一次推进只走一拍。
- **浏览器现场放**:接受它活在浏览器里,PPTX 只当备份。

## 7 · 一票否决(看到就回炉)
- 一环**说不出它解决的失败案例** → 砍掉或重想。
- **平铺直叙罗列**"我们有 A、B、C、D" → 改成"被问题逼出来"。
- **形容词堆砌 / 对仗排比上瘾** → 去 AI 味。
- **均匀递归、每环都讲到底** → 用深度预算砍。
- 只有炫技没 wow,或只有 wow 没深度 → 补另一半。
