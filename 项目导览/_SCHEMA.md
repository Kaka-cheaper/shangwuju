# 模块导览页 · 写作契约（每个模块兵必读）

你不是在写文档，是在写**导游词**。读者是第一次见这个项目的人，走完你这一站要能用自己的话复述这段主线。

## 产出方式
1. 复制 `_TEMPLATE.html` → 重命名 `module-<你的id>.html`。
2. **只改 `TOUR_DATA` 对象**。不改 `<head>` 引用、不写内联样式/脚本、不改 HTML 结构。
3. 样式/交互全部由 `assets/{tour.css,tour-engine.js}` 提供——这保证 7 页绝对一致。

## 硬规则（违反则破坏一致性或认知节奏）
1. **全程 6-10 步**（含骨架步）。`steps[0]` 必须 `kind:"skeleton"`：三段论 reveal + teach、**无 gap**、focus 是主线 2-3 个核心节点。
2. 其余步 `kind:"step"`，每步结构固定：**先 gap，后 reveal（一句话点破），再 teach（老师讲）**。
3. **gap 必须是真问题、必须含 `？`**——制造"我居然没想过"的缺口，不是过渡句。
   - ✗ "接下来看 critic。" ✗ "critic 怎么工作？"（太泛）
   - ✓ "LLM 出的方案把 5 岁娃排了 196 分钟爬山，谁来拦？"
4. **reveal = 一句话点破**（≤40 字，有因果方向），是给快速扫读者的"金句"。可用 `<b>`、`<code>`。
   - ✗ 罗列功能点 ✓ "下单先冻结、回调才扣减——两段式保证钱没到账货不会少。"
4.5 **teach = 字符串数组**，是这套导览**不像百科全书、像老师**的关键。每条渲染成一段，
   按 **「场景痛点 → 怎么办 → 妙在哪」三段式**，用『你/我们』对话口吻 + 具体例子，
   把"为什么聪明"讲透。**凡出现代码变量名（如 `planning_fast_path`），先用人话点明它是什么**，
   别裸抛术语（裸抛术语 = 百科全书腔）。
   - ✗ 把 reveal 换个说法重复一遍。✓ 先讲一个具体例子让人尝到痛，再揭做法，最后点破巧思。
5. **focus ≤ 3 个节点**（工作记忆 4±1 硬限制）。
6. **quiz 放 1-2 个**，放在"凭直觉会答错"处（条件分支/容错/异步/反直觉决策）。
   - 干扰项必须"合理但错误"；`wrong_note` 句式："你以为 X，实际 Y——因为 Z"。
   - 不出背诵题。
7. **每个 reveal 至少挂 1 条真实 `refs`**（`file` + `lines`），路径用**侦察兵核验过的当前路径**，
   **不要照抄 codesee features.json 的旧路径**（已大面积漂移）。引用前用 Read 打开文件核验行号。
8. 1-3 个最有张力的步配 `code`（折叠源码片段，真实粘贴、保留缩进，≤25 行）。
9. 收尾步落在"质量保障/兜底"类（critic / fallback / 容错），给读者"这系统是闭环的"安全感。

## 自查清单（写完逐条过）
- [ ] 把所有 gap 连起来读：是一条有悬念的故事线，还是目录？
- [ ] 把所有 reveal 连起来读：没见过项目的人能否复述出本模块主线？
- [ ] 每个 quiz 干扰项是"合理的错误直觉"还是凑数？
- [ ] 有没有哪步 reveal 涉及了 focus 之外的节点？（有就调 focus 或文案）
- [ ] 入场第一步是否接上了上一模块给你的"离场钩子"？结尾是否抛出了通往下一模块的钩子？
- [ ] 所有 refs 的 file:line 真实存在（已 Read 核验）？

## 字段速查
见 `_TEMPLATE.html` 内联注释。要点：
- `graph.nodes[].kind` ∈ `input | compute | decision | output | fallback`（决定节点点亮配色）。
- `graph.nodes[].col/row` 网格坐标（从 0 起），画出真实数据流方向；不填则竖排兜底。
- `meta.prev/next` 指向相邻模块文件，承载模块间"剧集钩子"。
- `code.lang` ∈ `py | js`；`code.lines[0]` = 源码起始行号（决定行号栏起点）。
